# Session Tree UI Specification

**Date:** 2026-01-11
**Status:** Scoping
**Approach:** Status-First with Progressive Disclosure

---

## Design Philosophy

> **Users don't care about tree structure. They care about:**
> 1. What's actively happening?
> 2. What needs my attention?
> 3. How do I get back to my main task?

The UI prioritizes **status visibility** over **structural hierarchy**.

**What "merged" means:**
- The parent session received and incorporated the child's results
- The child's work is visible in the parent conversation
- Deleting the child loses no information (it's redundant)

**What we keep:**
- Children still working (â— status)
- Children awaiting permission (ðŸ›¡ï¸ status)
- Children with unmerged results (parent hasn't acknowledged yet)

### Auto-Cleanup Implementation

```typescript
// Trigger 1: Idle timeout (runs on interval)
function startIdleCleanupTimer(instanceId: string, sessionId: string) {
  const IDLE_TIMEOUT = 5 * 60 * 1000 // 5 minutes

  setTimeout(() => {
    const session = getSession(instanceId, sessionId)
    if (!session) return // Already deleted
    if (session.parentId === null) return // Don't auto-delete parents
    if (session.status !== 'idle') return // Still working

    // Check if merged (parent has incorporated results)
    const isMerged = isSessionMergedIntoParent(instanceId, sessionId)
    if (isMerged) {
      deleteSession(instanceId, sessionId)
    }
  }, IDLE_TIMEOUT)
}

// Trigger 2: Parent activity (runs on message send)
function cleanupMergedChildren(instanceId: string, parentSessionId: string) {
  const children = getChildSessions(instanceId, parentSessionId)

  for (const child of children) {
    if (child.status !== 'idle') continue // Still working
    if (child.pendingPermission) continue // Needs attention

    const isMerged = isSessionMergedIntoParent(instanceId, child.id)
    if (isMerged) {
      deleteSession(instanceId, child.id)
    }
  }
}

// Called when user sends message in parent
function onUserMessageSent(instanceId: string, sessionId: string) {
  const session = getSession(instanceId, sessionId)
  if (session?.parentId === null) {
    // This is a parent session - cleanup merged children
    cleanupMergedChildren(instanceId, sessionId)
  }
}
```

### Determining "Merged" Status

A child is considered "merged" when:
1. Child status is `idle` (finished working)
2. Parent session contains a message referencing the child's completion
3. OR child has been idle for the full timeout period

```typescript
function isSessionMergedIntoParent(instanceId: string, childSessionId: string): boolean {
  const child = getSession(instanceId, childSessionId)
  if (!child?.parentId) return false

  // Simple heuristic: if child is idle, assume merged
  // (The parent wouldn't have gone idle without incorporating results)
  return child.status === 'idle'
}
```

### Thread State Aggregation

```typescript
function getThreadStatus(thread: SessionThread): Status {
  const statuses = [thread.parent, ...thread.children].map(s => s.status)

  // Priority order
  if (statuses.some(s => s === 'permission')) return 'permission'
  if (statuses.some(s => s === 'compacting')) return 'compacting'
  if (statuses.some(s => s === 'working')) return 'working'
  return 'idle'
}

function getThreadBadgeCount(thread: SessionThread): number {
  const status = getThreadStatus(thread)
  return thread.children.filter(c => c.status === status).length
}
```

## Open Questions

1. **Orphan Handling**: If user deletes parent but keeps children, how do we display orphaned sessions?
   - Option A: Promote first child to parent
   - Option B: Show in separate "Orphaned" section
   - Option C: Force delete children with parent

2. **Deep Nesting**: Do we support children of children?
   - Current recommendation: No, flatten to 2 levels max
   - Sub-agents of sub-agents become siblings

3. **Notification Center**: Should permissions also appear in a global notification area?
   - Current: Inline in dropdown
   - Alternative: Also show in header notification bell

4. **Auto-Expand**: When a child requests permission, should we auto-expand the dropdown?
   - Pro: User sees permission immediately
   - Con: Disruptive if user is focused elsewhere
